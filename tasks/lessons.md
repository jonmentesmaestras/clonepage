# Lecciones Aprendidas (Lessons Learned)

- **Iframe srcDoc y rutas relativas:** Al inyectar código HTML en un iframe utilizando el atributo `srcDoc`, las rutas relativas (como hojas de estilos CSS o imágenes locales) fallan porque el `baseURI` se toma del documento padre (en este caso, `localhost` o el dominio de desarrollo), no de la ruta original del documento clonado. Para solucionarlo, se debe inyectar dinámicamente una etiqueta `<base href="...">` temporal apuntando al directorio base del origen real (ej. bucket de S3) y removerla antes de guardar los cambios para no alterar el código fuente final. Adicionalmente, hay que tener en cuenta que las herramientas de scraping a menudo inyectan etiquetas `<base href="./index.html">` en el HTML descargado. Por lo tanto, no basta con inyectar la etiqueta si no existe, sino que hay que **sobrescribir temporalmente** cualquier etiqueta `<base>` existente, guardando su `href` original en un atributo `data-original-href` para poder restaurarla antes de guardar y evitar corromper la página.

- **Refactoring de template literals con código embebido:** Al refactorizar archivos que contienen template literals largos con código JavaScript embebido (como `iframeBridge.js`), **NUNCA usar ediciones parciales con replace_file_content**. Los escapes de backslashes (`\\n`, `\\w+`, `&&`, `>`) se corrompen al pasar por la herramienta de reemplazo, y la estructura de bloques `if/else if` se pierde fácilmente. La estrategia correcta es **reescribir el archivo completo con `write_to_file`** cuando el contenido es un template literal con código embebido. Esta lección se aplica a cualquier archivo que contenga strings multilínea con JavaScript dentro de JavaScript.

- **Persistencia de la etiqueta `<base>` en el Editor:** Al refrescar el editor manualmente (por ejemplo, después de guardar una traducción), no basta con inyectar el bridge. Se debe re-preparar el HTML capturando primero el `href` de la etiqueta `<base>` que apunta a S3 (el que permite ver los estilos en `localhost`) antes de limpiar el documento para S3. Si se inyecta en el editor el HTML limpio (que tiene el `<base>` original apuntando a `./index.html`), el iframe no podrá resolver los recursos y el diseño se romperá. El flujo correcto es: Capturar Base S3 -> Limpiar Documento -> Guardar en S3 con Base Original -> Re-preparar HTML para Editor con Base S3 -> Actualizar Store.
